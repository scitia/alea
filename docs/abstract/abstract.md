# The aleatoric execution chain design pattern as a non-deterministic decision-making process in server-side application modeling with synergy of business flow-oriented architecture

Today's enterprise applications model business processes using well-established market tools, technical concepts, and architectural approaches to deliver useful solutions to end users. However, the source code of such applications often focuses on implementing domain logic that is difficult to understand, which raises the entry barrier for new contributors and reduces project readability in terms of source code quality. This work presents a new architectural model based on decoupling the server-side application interface layer from business process definitions. These processes are defined as graphs composed of subprocesses represented as waypoints, thereby modularizing the application code responsible for domain logic. A separated module of the traditionally understood application interface integrates agentic tools that, based on data received from endpoints, decide which business paths to select and execute application source code by decomposing their process graphs. To realize this architectural concept, a programming framework was developed to support the composition and decomposition of business process graphs.
